Notes:
- `bootstrap.py` stays at the repository root intentionally so the human entrypoint is obvious.
- `compute_node/compute_methods/` owns the native implementations; `compute_node/performance_metrics/` measures and ranks them.
- `wire/` is now split by protocol role: discovery, internal control/data plane, and external control/data plane.
- On Windows, benchmark backend routing prefers the routine-safe CUDA path when an NVIDIA adapter is present.
- DX12 source remains in-tree for debugging, but its runtime and benchmark entry points are currently disabled.
- `compute_node/input_matrix/generated/` and `compute_node/performance_metrics/result.json`
  are local machine artifacts and should never be committed.
- DX12 build outputs are local-only by default; CPU/CUDA/Metal prebuilt runners may
  be tracked when intentionally refreshed.
